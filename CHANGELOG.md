# Changelog

## [1.9.0] - 2026-07-21

### Added

- Semantic frontend mapping with AST-backed components, routes, state, actions,
  ownership, imports, source hashes, and extraction provenance.
- Backend/API operation discovery and frontend parity analysis across Python and
  JavaScript/TypeScript applications.
- Interactive product-intent capture with field-level provenance, confidence,
  confirmation state, and map-derived fallback evidence.
- Source-aware redesign planning, structural proposal comparison, disposable
  prototype briefs, and runtime DOM/accessibility/layout evidence.
- Deterministic rendered findings for layout and font misalignment, clipped or
  intentionally truncated text and components, text edge contact, logical-axis
  padding imbalance, and inadequate line spacing across responsive viewports.
- Durable executable scan-to-redesign workflow state with freshness and blocker
  gates.

### Changed

- Split analyzer responsibilities into rule, AST, project, custom-check, and
  execution modules.
- Project memory now uses deterministic local JSON relevance matching, reducing
  base-install weight and dependency surface.
- Kept Pillow and Playwright behind optional capability extras so the base CLI
  installs only core analysis dependencies.
- Unified frontend file discovery and preserved project-root backend/API context
  during scoped scans.
- Hardened PyPI publication with release-only triggering, tag/version parity,
  distribution validation, and wheel smoke testing.

### Fixed

- Preserved backend parity during scoped frontend scans.
- Resolved root-relative asset lookup and JavaScript route extraction gaps.
- Added prompt-boundary isolation for repository-controlled content.
- Made machine-readable scan output clean and deterministic.
- Added Pillow to development dependencies so clean CI can collect capture
  tests.
- Replaced deprecated Pillow pixel iteration while retaining compatibility with
  Pillow 10 through current releases.
