        # Grill Gate

- Owner: repository operator.
- Problem: `uidetox setup` materializes generic inferred defaults, then later map-aware resolution treats those defaults as explicit and overwrites richer mapped intent.
- Out of scope: changing design dial defaults, redesign strategy selection, or existing explicit CLI values.
- Review failure: empty/default setup still suppresses mapped intent, explicit user values stop winning, old config files fail to load, or targeted tests fail.
- Riskiest assumption: explicit-vs-default provenance can be represented compatibly without a config migration command.
- Smallest acceptable: persist only genuinely explicit intent fields (or equivalent field-level provenance), resolve explicit → mapped inference → fallback defaults, and cover setup→persist→map→redesign behavior.
- Recommended choice accepted: preserve backward compatibility; infer explicit fields conservatively for legacy configs.

# Context

`DesignSettings.from_config` merges the complete configured dictionary over inferred intent. `commands/setup.py::run` calls it without a frontend map and writes the resulting generic `DesignIntent` back to config. A later redesign has a real semantic map, but configured generic fields win.

# Acceptance Criteria

- Intent resolution tracks field-level provenance: explicit user input, mapped inference, or fallback default.
- Explicit non-empty setup values win over mapped inference.
- Empty/default setup does not persist generic fields as explicit intent.
- Mapped `primary_job`, `genre`, `preserve`, and `constraints` survive an empty/default setup round-trip.
- Legacy configs without provenance remain loadable; non-default configured values remain explicit.
- `commands/next.py` and `commands/redesign.py` consume the same resolved intent policy.
- Tests reproduce the previous setup→persist→map failure and prove the fix.
- Existing serialized design-intent keys remain readable.

# Constraints

- No new dependency.
- Keep current CLI flags and design dial defaults.
- Preserve existing explicit setup behavior.
- Do not alter redesign strategy ranking in this spec.

# Review Notes

- Check empty strings and empty lists do not become explicit.
- Check legacy generic defaults do not mask inference.
- Check source/provenance metadata cannot override semantic fields accidentally.
