        # Grill Gate

- Owner: repository operator and agents using runtime maps, intent contracts, review, status, history, workflow, and finish gates.
- Problem: capture and runtime mapping launch independent browsers, pixel evidence has no semantic regions or source ownership, and downstream gates cannot distinguish fresh, stale, missing, or contract-relevant visual evidence.
- Out of scope: subjective visual scoring, generated reviewer boards, external uploaded baselines, and automatic dev-server startup.
- Review failure: duplicate browser paths remain, masks can hide arbitrary changes without provenance, stale evidence is treated as fresh, ownership/intent links are invented, or existing workflows require screenshots by default.
- Riskiest assumption: runtime element bounds and map ownership facts are stable enough to annotate exact pixel evidence without pretending the annotation is a quality judgment.
- Smallest acceptable: runtime observation becomes the single capture adapter, semantic/ignore regions are explicit manifest inputs, and all workflow consumers can enforce fresh evidence when explicitly requested.
- Recommended choice accepted: visual evidence stays opt-in by default; `--require-visual-evidence` is an explicit automation gate.

# Context

`runtime_observer.observe_frontend` already captures page screenshots and `RuntimeElement.bounds` in one browser session. `commands/capture.py` duplicates browser launch/navigation. Frontend maps already contain source hashes, runtime pages, ownership, intent provenance, and preserve contracts, but current diff metadata cannot reference them.

# Acceptance Criteria

- `uidetox capture` delegates browser observation to `runtime_observer` instead of maintaining a second Playwright lifecycle.
- Runtime observation accepts the minimum capture options needed by both `map --runtime` and `capture` without leaking command concerns into the adapter.
- Visual evidence accepts explicit semantic regions and explicit ignore regions derived from runtime bounds; clipping, overlap, and out-of-bounds behavior are deterministic and tested.
- Ignore regions carry reason and provenance and are serialized; there is no implicit global animation masking.
- Each changed semantic region reports stable identity, bounds, changed pixels/ratio, owning source targets when known, and relevant intent/preserve contract links when supported by map evidence.
- Manifest freshness validates source image hashes, configuration, runtime/map source hashes, and schema version; stale or incomplete evidence is never silently reused.
- Review, status, history, workflow, and finish expose missing/fresh/stale/blocked visual evidence consistently in text and JSON.
- CLI/config support tolerance, maximum pixels, dimension policy, color-management policy, evidence location, and `--require-visual-evidence` where the gate applies.
- Existing commands remain functional without Pillow or Playwright when visual evidence is not requested.
- Tests cover semantic masks, ignore provenance, ownership/intent links, stale source/config/map inputs, optional gate behavior, and runtime adapter reuse.

# Constraints

- Source facts and frontend map remain authoritative for ownership; do not guess file targets from selectors.
- Intent provenance remains field-level and must not be overwritten by visual evidence.
- `finish` may block on visual evidence only when the explicit requirement is enabled.
- Avoid circular imports between capture, runtime observer, frontend map, review, status, history, and workflow.

# Review Notes

- Trace browser launch sites and verify only the runtime observer owns Playwright.
- Verify ignored pixels are separately reported rather than removed from the audit trail.
- Verify freshness changes when any image, mask, relevant config, map source hash, or schema version changes.
- Verify no visual evidence is converted into a subjective design score.
